# REST API

- REST의 기분 원칙을 성실히 지킨 서비스 디자인을 RESTful이라고 표현

## REST API의 구성

- 자원, 행위, 표현의 3가지 요소로 구성됨

| 구성 요소 | 내용                           | 표현 방법        |
| :-------- | :----------------------------- | :--------------- |
| 자원      | 자원                           | URI(엔드포인트)  |
| 행위      | 자원에 대한 행위               | HTTP 요청 메서드 |
| 표현      | 자원에 대한 행위의 구체적 내용 | 페이로드         |

## REST API 설계 원칙

1. URI는 리소스를 표현해야 함 (동사보다는 명사 사용)
2. 리소스에 대한 행위는 HTTP 요청 메서드로 표현 (GET, POST, PUT, PATCH, DELETE)
