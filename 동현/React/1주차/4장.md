## 리액트의 특징

### 리액트의 탄생

리액트가 개발되기 전 다수의 기술은 정적인 웹 사이트를 구축하는 데 초점을 두고 있었다. 이에 페이스북 개발팀은 서비스의 변화가 많고 사용자와 상호작용이 원활한 대규모 웹 애플리케이션을 쉽게 구축할 수 있는 기술이 필요했고, 그렇게 탄생한 것이 리액트이다.

### 컴포넌트 기반의 유연성

리액트는 유연성이 있기 때문에 새로운 기능을 추가하거나 기능을 업그레이드 할 때 코드를 많이 수정하지 않아도 된다. 쉽게말해 복사 붙여넣기를 반복하지 않아도 된다는 말이다. 리액트는 모듈화를 사용해 이를 가능케 하였다. 여러 페이지에서 공통으로 사용하는 코드를 컴포넌트라는 단위의 모듈로 만들어 놓고 필요할 때 호출해서 사용한다.

### 쉽고 간단한 업데이트

리액트는 데이터의 변경이 생기면 불필요한 요소는 삭제하고, 새롭게 수정 사항을 반영한 요소를 다시 만들어 통째로 업로드한다.

### 빠른 업데이트

리액트는 정보의 갱신을 한번에 모아 수행하므로, 브라우저의 성능을 떨어뜨리지 않는다.

## 리액트 앱의 동작 원리

### 리액트 앱에는 어떻게 접속하는 걸까?

컴퓨터를 로컬 웹 서버로 사용하여 실행한다.
